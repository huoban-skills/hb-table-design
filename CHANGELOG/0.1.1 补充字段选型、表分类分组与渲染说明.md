## 详情

### 1. 新增: 字段类型选择最佳实践

「关联 vs 选项」判据只藏在"共享选项"一句里，建表蓝图想引用时无处可引。新增独立章节：

- **关联 vs 选项**（最关键）：判据是"枚举值是不是一个独立业务对象"，并列出"被多表引用 / 需附带属性 / 频繁增减 / 数量大"等命中条件。
- 另补：选项 vs 文本、成员 vs 文本、金额 vs 数值、关联 vs 字段冗余、子表 vs 关联、自增编号 vs 业务主键。

原「关联关系规范」的"共享选项"规则保留，作为关联结构的补充。

### 2. 新增: 表分类与分组最佳实践

表怎么分组没有规则。参考 ER 工具项目的分类与分组逻辑，新增：

- 四类：master 主数据 / base 基础数据 / transaction 业务表 / config 配置表。
- 关键判据：**被多张表引用 ≠ 基础数据**，看的是有没有明确业务归属。
- 分组规则：master 与同领域 transaction 同组、主子表两端必须同组、每组 2~7 张、base/config 按领域拆分不合并。

### 3. 修复: 统一表分类命名

四类中 `transaction`、`config` 的中文表述统一为「业务表」「配置表」（原"业务单据""配置参数"），与代码侧分类键保持对应。

### 4. 新增: 「渲染图片」节

`scripts/plantuml_render.py` 已存在，但 SKILL.md 没说明怎么用。新增「渲染图片」节并改写「输出要求」：

- 文档化用法：把 PlantUML 写入 `.puml`，渲染为 PNG/SVG（走官方服务器，自动注入统一样式，源文件不变）。
- 输出流程改为：先出代码 → 渲染成图 → 展示给用户。
