# 0.2.1 HTML 交互输出与样式统一

> 2026-06-30

HTML 相比 Markdown 可提供更好的阅读体验（排版、表格、图片一体呈现），同时不像 Markdown 需要 Typora 等专用工具才能正常阅读，双击即可打开。因此将 HTML 从可选导出升级为主输出物，并增加交互功能和样式统一。

## 摘要

1. 优化: HTML 成为主输出，输出流程改为 md → html 自动转换，不再询问是否导出
2. 新增: HTML 内嵌三个交互按钮——复制 PlantUML 代码、下载流程图 SVG、下载 ER 图 PNG
3. 优化: ER 图美化（品牌蓝表头、淡蓝分组背景），与流程图、HTML 共用一套视觉
4. 新增: 流程图每个环节标注对应的数据表（`table` / `tables`）

---

## 详情

### 1. 优化: HTML 成为主输出

此前 HTML 是可选导出项，md 完成后需询问用户。现在 HTML 是主输出物：`md_to_html.py` 在流程中自动执行，产出单文件 `表结构设计.html`（图片 base64 内嵌、无外部依赖），完成后自动打开。md 文件保留为源文件但不再是面向阅读者的成品。

### 2. 新增: 交互功能

`md_to_html.py` 新增三个能力：

- **复制 PlantUML 代码**：`er.puml` 以 base64 嵌入 HTML（`<script type="text/plain" id="puml-source">`），点击按钮解码并复制到剪贴板。兼容 `file://` 协议——`navigator.clipboard` 不可用时回退到 `document.execCommand('copy')`
- **下载流程图 / ER 图**：从已内嵌的 `<img>` data URI 直接创建下载链接，无需额外请求
- **按钮位置**：`_insert_action_buttons()` 用正则匹配 `<img alt="业务流程图">` 和 `<img alt="ER 图">`，在图片下方插入操作栏

### 3. 优化: ER 图美化

调整 `plantuml_render.py` 的 `ER_STYLE`：表头品牌蓝 `#3370FF` + 白字，分组容器淡蓝灰 `#F6F8FC`，边框柔和、箭头加粗变灰、加大节点间距。`_remove_header_gray()` 将 PlantUML 默认的 `#F1F1F1` 灰替换为白色。至此流程图、ER 图、HTML 三者视觉统一。

### 4. 新增: 流程图标注数据表

`flow_render.py` 的每个环节支持 `table`（单张）/ `tables`（多张）字段，卡片下行显示 `▸ 表名`，搭建者一眼看到「业务环节 ↔ 数据表」映射。主子表在同一个环节中合并展示（如"创建订单"标注 `["订单", "订单明细"]`）。
