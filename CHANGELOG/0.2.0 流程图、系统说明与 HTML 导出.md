## 详情

### 1. 系统说明（面向搭建者的施工概览）

此前文档第一屏就是 ER 图。新增置顶两节：

- **系统说明**：写给搭建者的施工概览，不是给终端用户的使用介绍。一段话交代业务背景与方案范围（几个模块、共几张表），再按模块列出每个模块包含哪些表、各表定位（主数据/业务表/基础数据）及承担的数据，并点出关键设计取舍（多表合并、关系实体化、外部对象抽表等）。
- **业务流程图**：L1 级主干、不画分支、按模块分组。

主文档结构现为：系统说明 → 业务流程图 → ER 图 → 表格清单 → 字段清单。

### 2. 业务流程图改用 SVG 生成（弃用 PlantUML）

PlantUML 活动图自动排版偏丑、且依赖公共服务器（实测会 509 限流）。新增 `scripts/flow_render.py`：读一份 JSON（分组 + 有序环节），纯标准库生成干净的 SVG——圆角卡片、分组容器配色、箭头串联，本地渲染无网络依赖。输出目录用 `flow.json`（源）+ `flow.svg`（图）替代原 `flow.puml`/`flow.png`。ER 图仍用 PlantUML。

### 3. md_to_html.py

环境无 `markdown` 库、无 `pandoc`，用纯标准库实现聚焦本 skill 输出子集的转换器：ATX 标题、管道表格、图片、引用、链接、加粗、列表、段落。图片（er.png / flow.svg）转 base64 内嵌，产物为单个 `.html`。
