## 详情

### 1. 新增: 表结构设计原则

0.1.x 只有选型和分组经验，缺少"为什么要拆表""什么时候该合表"等可检验的判断信号，AI 执行不稳定。新建 `reference/表结构设计原则.md`，三层十条 + 质量门，每条原则四件套格式（原则名/英文规范译名/信号/动作/例子）：

- **表级**（这张表该不该存在）：单一职责 Single Responsibility、主键约束 Primary Key Constraint
- **关系级**（表与表怎么连）：一对多分离 One-to-Many Decomposition、多对多解耦 Junction Table Decomposition、关系实体化 Relationship Promotion
- **字段级**（每个字段对不对）：字段原子化 Atomic Value Constraint、消除重复组 Column Group Elimination、字段对应实体 Dependency Alignment、字段类型匹配 Field Type Matching、计算替代录入 Computed over Manual

理论溯源至 1NF/2NF/3NF 和 ER 基数理论，附学术参考链接。

### 2. 新增: 表格清单

用户看到 ER 图后需要逐个翻字段清单才能了解每张表用途。在主文档结构中，ER 图和字段清单之间增加表格清单（表名/类型/用途），快速总览。

### 3. 优化: reference 拆分与 SKILL.md 精简

SKILL.md 流程、原则、类型表、分组规则混在一个文件，超过 360 行。将"依据什么做"的知识拆入 3 个 reference 文件，SKILL.md 只保留"怎么做"的流程：

- `reference/字段选型规则.md`：伙伴云 22+ 种字段类型表、选型判断规则、状态字段设计规则、关联字段命名约定
- `reference/表分类与分组.md`：4 类表定义、关键判据（"被多张表引用 ≠ 基础数据"）、6 条分组规则
- SKILL.md 从 ~368 行缩减到 ~240 行，只保留 Skill 定位、依赖关系、职责边界、执行流程、ER 图语法/模板、渲染、输出要求

### 4. 修复: 原则信号措辞

验证方案时发现多条原则的信号不够精确，导致漏检或误判：

- 单一职责：补充信号 ②"同一个现实对象会同时出现在多张表中（数据重叠）"，原版只覆盖结构重叠
- 一对多分离：英文从 Eliminate Repeating Groups 改为 One-to-Many Decomposition（准确对应关系级拆表）
- 关系实体化：信号从主观的"想记录"改为可验证的"配对本身需要记录独立于两端的属性（时间/数量/状态/附件）"
- 字段对应实体：信号从模糊的"在描述别的实体"改为 2NF/3NF 测试"删掉本表主键后，这个字段还有独立含义"
- 计算替代录入：从抽象的"派生替代"改为直白可理解的表述
