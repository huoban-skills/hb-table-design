# hb-er-draw

## 语法

用 `json` 定义表，`package` 按模块分组，箭头从父表（一）指向子表（多）：

```
@startuml

package "客户管理" {
  json 客户 {
    "字段名": "类型"
  }
}

客户 --> 联系人
客户 --> 线索 : 产生

@enduml
```

## 字段类型

类型统一用中文，从下表选取：

| 类型 | 说明 |
|---|---|
| 关联数据 | 关联另一张表的记录，如联系人关联所属客户 |
| 单行文本 | 名称、编号、地区等简短内容 |
| 多行文本 | 备注、跟进内容、需求描述等长文本 |
| 富文本 | 带排版的说明文档、产品介绍等 |
| 选项（A / B / C） | 枚举值，如客户状态（潜在/跟进中/已成交）；选项值写在括号内用 / 分隔 |
| 日期 | 创建时间、签署日期、预计成交日期等 |
| 时间 | 仅记录时刻，如预约时间、上班打卡时间 |
| 成员 | 负责人、跟进人、审批人等系统内用户 |
| 金额 | 合同金额、预估金额、付款金额等货币值 |
| 数值 | 数量、折扣率、库存等纯数字 |
| 开关 | 是否主联系人、是否启用、是否已读等布尔值 |
| 自增编号 | 记录需要独立编号以便引用或追溯，且无天然业务唯一标识时使用，如合同编号、工单编号、任务编号 |
| 评分 | 客户满意度、服务质量等星级评价 |
| 进度条 | 项目完成百分比、任务进度等 |
| 手机号码 | 联系人手机、客服电话等，支持格式校验 |
| 邮箱 | 联系邮箱，支持格式校验 |
| 附件 | 合同文件、报价单、资质证明等文档 |
| 图片 | 产品图片、现场照片、证件照等 |
| 位置 | GPS 定位坐标，如拜访签到、现场位置 |
| 地址 | 文字形式的地址，如公司地址、收货地址 |
| 手写签名 | 确认单、验收单等需要手写签字的场景 |
| 扫码 | 扫描商品条码、设备二维码等 |

## 关联关系规范

**业务主键**：优先用有意义的字段（如客户税号、统一社会信用代码）；需要独立编号以便引用或追溯、且无天然业务唯一标识时用 `自增编号`（如合同编号、工单编号）；无业务主键需求的表不画 ID 字段（如联系人、跟进记录）。

**1-n 外键**：N 表必须有一个 `关联数据` 字段，命名为 **所属xx**（xx 为父表名）。

**共享选项**：若同一组选项值被多张表引用，需将其抽取为配置表，引用方改用 `关联数据` 关联该配置表。配置表不在 ER 图中以 `json` 实体出现，仅体现为关联字段的类型标注。

```
json 联系人 {
  "所属客户": "关联数据",
  "姓名": "单行文本",
  "手机": "手机号码"
}
```

## 完整模板

```
@startuml

package "客户管理" {
  json 客户 {
    "客户税号": "单行文本",
    "客户名称": "单行文本",
    "所属行业": "选项（互联网 / 制造业 / 金融 / 其他）",
    "客户状态": "选项（潜在 / 跟进中 / 已成交 / 已流失）",
    "负责人": "成员",
    "创建时间": "日期"
  }
  json 联系人 {
    "所属客户": "关联数据",
    "姓名": "单行文本",
    "手机": "手机号码",
    "邮箱": "邮箱",
    "是否主联系人": "开关"
  }
}

package "销售管理" {
  json 线索 {
    "所属客户": "关联数据",
    "来源渠道": "选项（官网 / 转介绍 / 广告投放 / 展会 / 其他）",
    "线索状态": "选项（待跟进 / 跟进中 / 已转化 / 无效）",
    "负责人": "成员",
    "创建时间": "日期"
  }
  json 商机 {
    "所属客户": "关联数据",
    "所属线索": "关联数据",
    "商机类型": "选项（新客户 / 续费 / 增购）",
    "预估金额": "金额",
    "销售阶段": "选项（初步接触 / 需求确认 / 方案报价 / 谈判中 / 已赢单 / 已输单）",
    "负责人": "成员",
    "预计成交日期": "日期"
  }
  json 合同订单 {
    "合同编号": "自增编号",
    "所属客户": "关联数据",
    "所属商机": "关联数据",
    "合同类型": "选项（新签 / 续签 / 补充协议）",
    "合同金额": "金额",
    "合同状态": "选项（草稿 / 审批中 / 已签署 / 已到期 / 已终止）",
    "签署日期": "日期",
    "到期日期": "日期"
  }
}

客户 --> 联系人
客户 --> 线索
线索 --> 商机
商机 --> 合同订单

@enduml
```

## 输出要求

直接输出可粘贴的代码块，代码后用不超过 3 行说明模块划分和主要关系。
