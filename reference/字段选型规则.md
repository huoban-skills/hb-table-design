# 字段选型规则

本文用于确认每个字段该选什么类型，从字段类型目录中选取，再按选型规则逐条对照。

## 字段类型目录

类型统一用中文，从下表选取：

| 类型            | 说明                                            |
| ------------- | --------------------------------------------- |
| 关联数据          | 关联另一张表的记录，如联系人关联所属客户                          |
| 单行文本          | 名称、编号、地区等简短内容                                 |
| 表单说明          | 表单内的分组标题行，本身不存储数据，仅作为分组分隔符                    |
| 多行文本          | 备注、跟进内容、需求描述等长文本                              |
| 富文本           | 带排版的说明文档、产品介绍等                                |
| 选项（A / B / C） | 枚举值，如客户状态（潜在/跟进中/已成交）；选项值写在括号内用 / 分隔          |
| 日期            | 创建时间、签署日期、预计成交日期等                             |
| 时间            | 仅记录时刻，如预约时间、上班打卡时间                            |
| 成员            | 负责人、跟进人、审批人等系统内用户                             |
| 金额            | 合同金额、预估金额、付款金额等货币值                            |
| 数值            | 数量、折扣率、库存等纯数字                                 |
| 开关            | 是否主联系人、是否启用、是否已读等布尔值                          |
| 自增编号          | 记录需要独立编号以便引用或追溯，且无天然业务唯一标识时使用，如合同编号、工单编号      |
| 评分            | 客户满意度、服务质量等星级评价                               |
| 进度条           | 项目完成百分比、任务进度等                                 |
| 手机号码          | 联系人手机、客服电话等，支持格式校验                            |
| 邮箱            | 联系邮箱，支持格式校验                                   |
| 附件            | 合同文件、报价单、资质证明等文档                              |
| 图片            | 产品图片、现场照片、证件照等                                |
| 位置            | GPS 定位坐标，如拜访签到、现场位置                           |
| 地址            | 文字形式的地址，如公司地址、收货地址                            |
| 手写签名          | 确认单、验收单等需要手写签字的场景                             |
| 扫码            | 扫描商品条码、设备二维码等                                 |

## 类型选择原则

逐字段选类型：先定主键标识，再按语义选基础类型，最后判断是否用关联替代。

### 主键设计

| 动作 | 信号 | 例子 |
|------|------|------|
| 增加自增编号 | 无天然唯一标识，需独立编号 | 合同编号、工单编号 |

### 类型选择

| 动作 | 信号 | 例子 |
|------|------|------|
| 用数值替代文本 | 语义是数量/金额/指标 | 库存数量 → 数值，单位写备注 |
| 用选项替代文本 | 值可枚举且固定 | 优先级 → 选项 |

### 用关联字段扩展

| 动作            | 信号                | 例子                |
| ------------- | ----------------- | ----------------- |
| 用关联替代选项       | 枚举值是独立业务对象（见下方判断） | 供应商 → 关联；优先级 → 选项 |
| 用关联替代文本       | 值重复出现且对象有自身属性     | 客户名多处出现 → 关联客户表   |
| 用关联替代成员       | 引用外部人员，非系统内用户     | 供应商联系人 → 建表 + 关联  |
| 用附加关联替代冗余字段   | 子表要显示父表信息         | 订单显示客户电话 → lookup |
| 用子表替代关联       | 主从关系，随主表增删        | 订单明细 → 订单的子表      |
| 关联字段命名为"所属xx" | 1-n 外键，N 表字段      | 订单明细.所属订单         |

怎么判断"是不是独立业务对象， 用关联替代选项？命中任一条就用关联：

| 判断 | 用关联 | 用选项 |
|---|---|---|
| 同组值被**多张表**引用 | ✅ 抽配置表 | ❌ 各表维护一份会不一致 |
| 需要**附带属性** | ✅ 关联到独立表 | ❌ 选项只能存一个名字 |
| 值会**频繁增减** | ✅ 增减即加数据行 | ❌ 改选项要动字段配置 |
| 值**数量大**（几十上百） | ✅ 关联 | ❌ 列表过长难维护 |
| 都不命中 | — | ✅ 选项最简 |
