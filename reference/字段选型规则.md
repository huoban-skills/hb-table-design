# 字段选型规则

伙伴云平台字段类型体系及选型判断规则。

## 字段类型

类型统一用中文，从下表选取：

| 类型 | 说明 |
|---|---|
| 关联数据 | 关联另一张表的记录，如联系人关联所属客户 |
| 单行文本 | 名称、编号、地区等简短内容 |
| 表单说明 | 表单内的分组标题行，本身不存储数据，仅作为分组分隔符 |
| 多行文本 | 备注、跟进内容、需求描述等长文本 |
| 富文本 | 带排版的说明文档、产品介绍等 |
| 选项（A / B / C） | 枚举值，如客户状态（潜在/跟进中/已成交）；选项值写在括号内用 / 分隔 |
| 日期 | 创建时间、签署日期、预计成交日期等 |
| 时间 | 仅记录时刻，如预约时间、上班打卡时间 |
| 成员 | 负责人、跟进人、审批人等系统内用户 |
| 金额 | 合同金额、预估金额、付款金额等货币值 |
| 数值 | 数量、折扣率、库存等纯数字 |
| 开关 | 是否主联系人、是否启用、是否已读等布尔值 |
| 自增编号 | 记录需要独立编号以便引用或追溯，且无天然业务唯一标识时使用，如合同编号、工单编号、任务编号 |
| 评分 | 客户满意度、服务质量等星级评价 |
| 进度条 | 项目完成百分比、任务进度等 |
| 手机号码 | 联系人手机、客服电话等，支持格式校验 |
| 邮箱 | 联系邮箱，支持格式校验 |
| 附件 | 合同文件、报价单、资质证明等文档 |
| 图片 | 产品图片、现场照片、证件照等 |
| 位置 | GPS 定位坐标，如拜访签到、现场位置 |
| 地址 | 文字形式的地址，如公司地址、收货地址 |
| 手写签名 | 确认单、验收单等需要手写签字的场景 |
| 扫码 | 扫描商品条码、设备二维码等 |

## 选型规则

### 关联 vs 选项（最关键）

一组枚举值，到底用 `选项` 还是抽成配置表用 `关联数据`？按以下任一条命中就用**关联**：

| 判断 | 用关联 | 用选项 |
|---|---|---|
| 同组值被**多张表**引用 | ✅ 抽配置表，多表关联同一张 | ❌ 各表各维护一份会不一致 |
| 选项项需要**附带属性**（供应商有联系人、地址） | ✅ 关联到独立表 | ❌ 选项只能存一个名字 |
| 值会**频繁增减**、由业务自己维护（商品、项目、部门、客户） | ✅ 关联，增减即加数据行 | ❌ 改选项要动字段配置 |
| 值**数量大**（几十上百） | ✅ 关联 | ❌ 选项列表过长难维护 |
| 固定、少量、无附加属性（状态、是否、优先级、性别） | — | ✅ 选项最简 |

> 一句话：**枚举值"是不是一个独立业务对象"**——是（供应商、商品、部门）就用关联，纯标签（待处理/已完成）就用选项。

### 其他选型规则

- **数值 vs 文本**：字段语义是数量、金额、统计指标等数字，类型用`数值`，不论来源数据是否带有单位缩写或格式。单位在备注中标注。
- **选项 vs 文本**：值可枚举 → `选项`；自由填写、无固定集合 → `单行文本`。
- **成员 vs 外部用户**：系统内用户（负责人、审批人）→ `成员`；外部人员（外部合作方、供应商联系人等）→ 建立**外部用户表**，用 `关联数据` 关联，不能用 `单行文本` 存名称。外部用户表至少包含姓名、手机/邮箱等联系字段，被多张表引用时共用同一张表。
- **关联 vs 文本**：某字段的值在多条记录中重复出现、且对应对象有自身属性需要管理，应先抽成主数据表再建关联，不能用文本存名称。
- **关联 vs 字段冗余**：子表要显示父表信息时，用 `关联数据` + 附加显示字段（lookup），不要在子表重复存父表字段。
- **子表 vs 关联**：主从、随主表一起增删 → 子表标签页；独立实体的多对一 → `关联数据` 字段。
- **自增编号 vs 业务主键**：有天然唯一标识 → 用该业务字段；需独立编号供引用追溯 → `自增编号`；无引用需求 → 不建 ID 字段。
- **共享选项**：同一组选项值被多张表引用时，抽取为配置表，引用方改用 `关联数据`。配置表不在 ER 图中以 `json` 实体出现，仅体现为关联字段的类型标注。

### 状态字段设计规则

- 状态值要覆盖每个中间阶段，不能只有起止状态。有流水线步骤的业务，每个可等待、可暂停的环节都应有对应状态。

### 关联字段命名

- 1-n 外键：N 表的关联字段命名为 **所属xx**（xx 为父表名）
