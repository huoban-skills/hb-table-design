# 表结构设计原则

## 一、表格

>判断这张表该不该存在

| 原则   | 规范译名                   | 信号                                  | 动作                  | 例子                                                       |
| ---- | ---------------------- | ----------------------------------- | ------------------- | -------------------------------------------------------- |
| 主键约束 | Primary Key Constraint | 找不到一个字段能唯一指认一行                      | 指定或新增唯一标识，可用复合键     | 客户用税号；跟进记录用 客户+时间+跟进人                                    |
| 单一职责 | Single Responsibility  | 有两张表结构或含义相同；同一个现实对象会同时出现在多张表中（数据重叠） | 合并成一张，差异变字段或角色/类型字段 | 基础/重点/战略客户 → 一张客户表 + 等级字段；词曲作者和外部协作人都是外部合作人 → 一张表 + 角色字段 |

## 二、表格关系

>判断表与表怎么连

| 原则    | 规范译名                         | 信号                            | 动作                    | 例子                        |
| ----- | ---------------------------- | ----------------------------- | --------------------- | ------------------------- |
| 一对多分离 | One-to-Many Decomposition    | 一条 A 对应好几条 B                  | B 拎成新表，B 表加指回 A 的关联字段 | 一个订单多个产品 → 订单明细表 + "所属订单" |
| 多对多解耦 | Junction Table Decomposition | A 能配多个 B，B 也能配多个 A            | 中间建一张关联表连两边           | 产品 × 仓库 → 库存表             |
| 关系实体化 | Relationship Promotion       | 配对本身需要记录独立于两端的属性（时间/数量/状态/附件） | 配对独立建成业务表             | 学员 × 作业要记提交时间和附件 → 提交记录表  |

## 三、表格字段
>判断每个字段对不对

| 原则     | 规范译名                     | 信号                  | 动作                      | 例子                    |
| ------ | ------------------------ | ------------------- | ----------------------- | --------------------- |
| 字段原子化  | Atomic Value Constraint  | 一格里塞了多个信息           | 拆成多个独立字段                | "红色/XL/纯棉" → 颜色、尺码、材质 |
| 消除重复组  | Column Group Elimination | 出现 字段1、字段2、字段3 横向编号 | 竖过来拆子表（触发关系原则 3）        | 产品1/产品2/产品3 → 订单明细表   |
| 字段对应实体 | Dependency Alignment     | 删掉本表主键后，这个字段还有独立含义  | 移到它该在的表                 | 订单表里的"产品材质" → 产品表     |
| 字段类型匹配 | Field Type Matching      | 按语义选字段类型            | 见 `reference/字段选型规则.md` | 独立业务对象用关联，纯标签用选项      |
| 计算替代录入 | Computed over Manual     | 值能由其他字段算出来          | 设为公式字段，不手填              | 天数 = 成品日期 − 开始日期      |

## 四、质量门（生成后校验）

| 检查项   | 信号                 | 动作                     |
| ----- | ------------------ | ---------------------- |
| 孤立表检测 | 某张表无任何关联（进或出）      | 判断：补关联 / 废弃清理 / 不属于本系统 |
| 命名规范  | 出现"数据表1""备份"等无意义命名 | 改为业务语义命名，同类表风格统一       |

## 理论溯源

- 表级：3NF 单一职责 + 1NF 主键要求
- 关系级：ER 基数（一对多 / 多对多 / 关系属性）
- 字段级 1~3：1NF 原子性 + 1NF 消除重复组 + 2NF/3NF 函数依赖
- 字段级 4~5：平台规范（字段选型规则）
