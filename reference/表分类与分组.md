# 表分类与分组

## 表分类（4 类）

先给每张表定类别，再按规则归组。

| 类别       | 含义                 | 例子              |
| -------- | ------------------ | --------------- |
| **主数据**  | 有明确业务归属的核心实体       | 客户、供应商、产品、项目、员工 |
| **基础数据** | 无明确业务归属、被各模块通用引用   | 单位、币种、部门、仓库、地区  |
| **业务表**  | 有生命周期的业务事件/操作记录    | 订单、入库单、发票、审批单   |
| **配置表**  | 低频的系统配置、审批流模板、状态枚举 | —               |

> **关键判据：被多张表引用 ≠ 基础数据。** 看的是"有没有明确业务归属"。
> 客户被销售单、合同、回款单引用，但归属是客户/销售管理 → 是**主数据**；
> 单位被采购、销售、库存引用，没有归属方 → 才是**基础数据**。

## 分组规则

- 按业务模块分组，**同一张表尽量只进一个组**，组名控制在 4 字内
- **主数据放进它的核心业务组**，与该领域的业务表同组（客户 → 客户管理，与销售单同组）
- **主子表关系两端必须同组**：业务表多对一引用主数据/业务表时不可拆开
- **单据流转关系**（业务表 → 业务表，上下游单据）可同组也可跨组
- **每组 2~7 张表**：少于 2 张不单独成组，超过 7 张按流程阶段或子领域拆
- **基础数据/配置表按领域拆成多组**（组织架构 / 地区行政 / 财务参数 / 产品基础 / 仓储配置 等），**不要合并成一个大组**
